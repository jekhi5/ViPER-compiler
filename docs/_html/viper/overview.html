Language Overview

ViPER is a dynamically-typed scripting language. Types are only determined at runtime.

ViPER's syntax is simple. Indentation is not meaningful. Line comments are marked with #.

Assignments

Names in ViPER are immutable. Once a name is given a value, that relationship cannot change. The name is called an identifier, and the entire expression is called an assignment.

Assignments use the let...in form. Identifiers may only include alphanumeric characters and underscores, and may not start with a number.

    let <indentifier> = <value> in <result>
    let foo = 1 in foo + 2

    3

Multiple assignments can be provided in a single let...in form, separated by commas.

    let
        a = 1,
        b = 2,
        c = 3
    in
        a + b - c
    
    0

Identifiers are immutable. Assignments within inner can shadow indentifiers in outer scopes.

    let
        foo = 1
    in
        let
            foo = 2
        in
            foo
    
    2

All assignment in a single let...in are evaluated independently. This means that they cannot "see" each other. For recursive bindings, use the rec keyword:

    let rec
        a = 1,
        b = a + 1,
        c = b + 2
    in
        c
    
    4

Multiple identifiers can be bound at once, based on the structure of a tuple. This is sometimes called tuple unpacking. The left-hand-side of the assignment must match the structure of the tuple on the right-hand-side.

    let t = (1, 2, 3) in
    let a, b, c = t in
    print(a);
    print(b);
    print(c)

    1
    2
    3

Note that in the case of nested tuples, these assignments can be as deep as desired:

    let t = (1, (2, (3, 4))) in
    let a, (b, (c, d)) = t in
    d

    4

... or as shallow:

    let t = (1, (2, (3, 4))) in
    let a, b = t in
    b

    (2, (3, 4))

An _ is used to signify an assignment where the name does not matter. The bound expression will still be evaluated.

    let
        _ = print(2 + 2)
    in
        1
    
    4
    1

This is especially useful in the case of tuple assignments, where some elements of the tuple may be irrelevant.

    let t = (3, ((4, true), 5)) in
    let x, ((y, _), z) = t
    x + y + z

    9

Control Flow

Control flow in ViPER is rather limited. The two main forms are conditionals and sequencing. ViPER also supports exception handling, discussed in a different section.

Conditionals

ViPER's only conditional is the if form:

    if <test-expr>:
        <then-expr>
    else:
        <else-expr>

The test-expr is evaluated first. If it evaluates to true, the then-expr is evaluated next. If not, the else-expr is evaluated. Note that the only truthy value is true, which is different from many other programming languages.

    if 5 > 3: 42 else: -1

    42

For multiple checks, nesting is required:

    let age = input() in
    # Determine which discount code to apply
    if age < 13:
        1
    else: 
        if age >= 18 && age <= 22:
            2
        else:
            if age > 65:
                3
            else:
                0

Sequencing

Sometimes it is useful to evaluate multiple expressions in sequence without caring about the returned values. The ; operator allows this. x`

    1 + 1; 2 + 2; 3 + 3

    6

This is most useful for functions with side effects, such as IO interactions and tuple mutation.

    print(1 + 1); print(2 + 2); print(3 + 3)

    2
    4
    6
    6

Long sequences can be formalized with the begin...end form, but this is purely decorational:

    begin
        print(1 + 1); 
        print(2 + 2);
        print(3 + 3)
    end

    2
    4
    6
    6
